DESCRIPTION

Till is an easy to use project-oriented templating front-end, with built-in POM metadata support.

FEATURES

  • Easy to use.
  • Uses Tilt. Easy.
  • Uses POM metadata. Easy.
  • Templates go where the file goes. Easy.
  • Did I mention it was easy?

SYNOPSIS

Till supports two types of templating, traditional whole file templating and inplace/inline templating.

All templates provide POM metadata in the rendering context.

Whole File Templating

Whole file templating takes a designated template, renders it according to specified filters and then save it to a designated output.

Till template files require an extension of .til or .till . Till templates can have YAML front matter to specifiy there filter and output options. Here is an example README.til file.

---
output: README.rdoc
format: rdoc
filter: erb
---

= <%= title %>

<%= title %> is such a wonderful project.

Copyright <%= released.strftime("%Y") %> <%= author %>

If no YAML front matter is present, then the template will be rendered via Erb to a file less the .til or .till extension. Eg. README.til would be processed by Erb to produce README.

Because Till use Tilt, a number of filtering options are available. There are three types of filters: formatters, stencils and miscellaneous. Formatters handle layout and depend on the format of the given document. A good example of such a filter is html which converts the given format, eg. markdown, to html. A stencil filter, on the other hand, handles dynamic content, interpolating variable substituations. erb, liquid and mustache are all examples of supported stencils. Miscellaneous filters handle odd-and-end transforms, for instance a custom filter to mask curse words.

Inline Templating

Inline, or inplace, templating makes it possible to add dynamic rederings to otherwise static file, without the need for a separate template file. This if acheived by utilizing file types that support comments. Currently only Ruby is supported.

Inline templating is indicated by the :till: marker. For example, in a Ruby script we might add an inline till comment as follows:

  module YourLibrary
    VERSION = "1.0.0"  #:till: VERSION = "<%= version %>"
  end

As given, an inline template will replace the whole line when rendered (albiet honoring indention). We can use the ^ match carrot to indicate that the rendering should only replace the line from the matching text on. Notice also in this example that we can template comments as well.

  # Manifest generated on XXXX  #:till: ^on <%= Time.now %>

Multi-line templating can be acheived by specifying the number of lines to absorb.

  # :till+3: Dir['*'].join("\n")
  file1
  file2
  file3

Notice the +3. This indicates that the next three lines (after any additional commented code lines) are to be replaced by this rendering.

Running the till Command

  $ till **/*.rb **/*.till

All .till files and all #:till: inline templates will be filled in.

Front matter options can be overridden on the command-line.

HOW TO INSTALL

To install with RubyGems simply open a console and type:

  $ gem install till

Local installation requires Setup.rb (gem install setup), then download the tarball package and type:

  $ tar -xvzf till-1.0.0.tar.gz
  $ cd till-1.0.0.tar.gz
  $ sudo setup.rb all

Windows users use ‘ruby setup.rb all’.
